## Current Findings

- The June 18 recipient evidence card is still the right public worksheet, but
  June 19 adds a privacy rule: raw candidate source URLs should not appear in
  Git when they can identify a real lab, CRO, provider, person, or business
  route.
- Public recipient cards should use an alias, `source_bundle_ref`,
  source-kind labels, source-count labels, capability labels, constraints, and
  a decision label. The private source index outside Git carries real names,
  URLs, contacts, and source-to-capability mapping.

  thalassemia and gamma-globin context for endpoint specificity, HUDEP2/HbF and

  for safety-capability labels.
- HHS de-identification guidance and the RIKEN HUDEP-2 page point to the same
  operating rule: a source can be useful for auditability while still being too
  identifying for a public candidate card.

  clinically routed.
