## Current Practical Goal

Use one of these public-safe follow-up states:

- `follow_up_not_due`;
- `follow_up_ready`;
- `follow_up_sent`;
- `records_pending`;
- `owner_redirected`;
- `alternate_owner_needed`;
- `response_captured_private`;
- `route_closed_label_only`;
- `release_blocked`.

## Current Blockers

- No raw owner communication can enter the public repo.
- A missing reply is not a biomedical finding.
- A redirect returns to specialist-owner routing.
- A record request returns to the private source index and record request
  matrix.
- A substantive reply returns to owner response capture.
- Patient-specific treatment instructions remain private and clinician-owned.
- Diagnosis wording, molecular subtype, transfusion burden, immune/blood-bank
  risk, iron/organ risk, and advanced-therapy relevance remain blocked until
  the right owner closes that domain.

## Strongest Research Lanes

1. **Owner follow-up discipline.** Use one state, one owner route, and one
   narrow question for each follow-up.
2. **Owner-response capture.** If a reply arrives, convert it into a private
   capture row, answer type, public-safe label, and next action.
3. **Private record indexing.** Keep raw records and raw communication outside
   Git; publish only de-identified labels after release checks.
4. **Transfusion-burden readiness.** Complete the private minimum fields before
   calculating annual `ml/kg/year`, red-cell volume, hemoglobin response, or
   iron input.
5. **Immune and blood-bank packet.** Keep antibody, DAT/direct Coombs,
   matching, reaction, spleen, and hemolysis questions routed to the right
   owner.
6. **Iron and organ-risk packet.** Keep ferritin, LIC MRI, cardiac `T2*`,
   chelator monitoring, and organ screening together before any specialist
   interpretation.
7. **Specialist-screening readiness.** Treat HSCT, gene therapy, CRISPR
   therapy, luspatercept, mitapivat, hydroxyurea, and trial questions as
   screening lanes, not repository recommendations.

## Patient-Context Data Still Needed

Collect raw records only in ignored private storage. Public updates should stay
de-identified and label-level unless the release checklist passes.

- follow-up state: owner route, owner role, outreach alias, follow-up alias,
  generalized timing, current state before follow-up, narrow question used,
  missing-record category, next owner role, response-capture alias, and release
  decision;
- owner response capture: source alias, domain answered, answer type,
  missing-record category, next owner role, public output label, and release
  decision;
- diagnosis and molecular context: current clinician diagnosis wording,
  clinical-dependence category, HPLC or electrophoresis timing, `HBB`,
  `HBA1/HBA2`, HPFH, delta-beta-thalassemia, HbF-modifier, and family-testing
  context if available;
- transfusion-burden minimums: date, body weight, transfused volume or approved
  unit conversion, red-cell fraction or unit hematocrit, pre-Hb, post-Hb if
  available, at least two rows, and at least a 28-day window;
- immune and blood-bank packet: antibody screen, named alloantibodies,
  DAT/direct Coombs specificity, extended matching policy, red-cell
  phenotype/genotype, crossmatch difficulty, reaction history, spleen context,
  and hemolysis workup;
- iron, chelation, and organ-risk packet: ferritin trend, LIC MRI, cardiac
  `T2*`, chelator identity, monitoring, side effects, and organ-risk screening;
- advanced-therapy history: prior discussion, referral, rejection, access
  barrier, or specialist opinion for HSCT, gene therapy, CRISPR therapy,
  luspatercept, mitapivat, hydroxyurea, or trial screening.
